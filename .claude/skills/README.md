# .claude/skills/

**Generated directory.** Entries are symlinks to `.agents/<skill>/`, created by
`scripts/agents/setup-cross-agent-skills.ps1` (Windows) or `.sh` (POSIX).

## Editing skills

Edit `.agents/<skill>/SKILL.md` directly. **Do not edit files here** — they are
symlinks; changes apply to the underlying `.agents/` file, but the directory
layout itself is regenerated by the setup script and any non-symlink, non-
allowlisted entries are removed.

## Adding or removing a skill

1. Add or remove `.agents/<skill>/SKILL.md`.
2. Re-run `scripts/agents/setup-cross-agent-skills.ps1` (or `.sh`).

A `post-merge` git hook re-runs the setup automatically when `.agents/` or the
setup scripts change in a pull/merge.

## Deactivated skills

Skills retired but kept for reference live as `.agents/<name>/REFERENCE.md`
(no `SKILL.md`). They are ignored by Claude Code, GitHub Copilot, and the
Codex plugin.

## Why three locations?

The active SKILL.md lives in `.agents/<skill>/`. The setup script projects it
to three tool-specific locations:

| Tool          | Location                                   | Mechanism   |
| ------------- | ------------------------------------------ | ----------- |
| Claude Code   | `.claude/skills/<skill>`                   | symlink     |
| GitHub Copilot| `.github/skills/<skill>`                   | symlink     |
| Codex plugin  | `.agents/plugins/plugins/ahkflowapp/skills/<skill>/SKILL.md` | hard link |

Codex uses hard links because its plugin installer dereferences symlinks
incorrectly. The post-merge hook exists primarily to repair hard links broken
by `git pull` replacing the source inode.
