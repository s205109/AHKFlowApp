# Hotkeys store a typed Action, with Raw as the only verbatim path

A hotkey's Action is persisted as a discriminated set of typed columns — an `ActionKind` plus the zero, one or two columns that kind owns (`Text`, `SendKeysContent`, `RunTarget` + `RunTargetKind`, `WindowOp`, `RemapDest`, `Body`; `Disable` owns none) — rather than one free string or a JSON payload. The kind is what the validator, the emitter, the DTOs and the history snapshot all branch on: each kind declares which field it requires, forbids the others, and has exactly one emission rule. Storage is gated by `HotkeyRules` on the create and update path: an `ActionKind` outside the enum is rejected outright, the kind's own field is required, and every other kind's is refused — so an Action accepted there is one the emitter can generate. Restore and Revert are the exception. `LegacyHotkeySnapshotConverter.ToDefinition` passes a W1 snapshot's typed fields through unvalidated, so a `Window` row with a null `WindowOp` is storable and throws only at generation time. The emitter throwing is a last line of defence, not the gate.

That is what closes the injection hazard the feature carried (issue #195), and it closes it on both sides of the `::`. The left-hand side is never free text: modifiers are booleans and the key is a canonical registry name or a `vkNN`/`scNNN` code, both validated at the create and update boundaries. On the right, the three kinds that embed user text (`SendText`, `SendKeys`, `Run`) pass it through the shared `AhkEscaping.EscapeStringLiteral`; `SendKeys` and `Remap` additionally persist a token validated against the key registry; `Window` emits from an enum and `Disable` emits a fixed word. The guarantee is deliberately one-sided: no Action can inject into the left-hand side, but a Raw `Body` can break out of the line it is emitted on — that is the escape hatch the next paragraph describes.

**Raw is the sole verbatim path, and it is not sandboxed.** Its `Body` is emitted exactly as stored — no wrapper, no escaping, so a block body carries its own braces. Validation checks shape only: non-empty, balanced `{`/`}`, no line starting with a `#` directive, no stray control characters. `\n`, `"` and `;` are all permitted, so a body of `return` followed by a second line `^a::Run("calc")` passes and injects a whole new top-level definition. AutoHotkey also parses the whole file at load, so a syntax error in one Raw body aborts the **entire** Profile script rather than just its own binding, and the brace-balance check has no awareness of string literals or comments, so a body can close its block early and have its remainder parsed at top level. ADR-0002 took the same position for Raw hotstrings — Raw exists precisely for what structured fields cannot express, so its text is kept as the source of truth rather than decomposed — and the naive brace count is the trade-off recorded under *Known limitations* in `docs/development/ahk-v2-syntax.md`: a string- and comment-aware scanner would drift toward being a script IDE. Raw must therefore be presented as unchecked AutoHotkey wherever it is offered.

## Migrating off the legacy pair

The typed columns replaced a two-value `HotkeyAction` plus one opaque `Parameters` string, in two migrations rather than one: the first adds the typed columns and back-fills them, a later one drops the legacy pair, so every commit in between runs against a database of either shape. The back-fill is hand-written T-SQL and the identical transform exists in C# as `LegacyHotkeyDefinitionConverter`, because history JSON written before the change still has to be read on Restore and Revert; a Testcontainers parity test runs the same fixtures through both and requires identical output.

The mapping preserves what the app already emitted, with one deliberate exception. `Run` becomes `Run`, and every `Send` whose parameters do *not* parse as a valid key token becomes `Raw` with a body reproducing the previously emitted `Send("…")` line byte for byte — which is the reason Raw emits verbatim instead of wrapping the body in braces. Those two branches regenerate identically.

A `Send` whose parameters do parse as a valid key token becomes `SendKeys`, and that branch does change: `SendKeys` now carries the auto `$` prefix the pre-W1 emitter never wrote, so `^!v::Send("^v")` regenerates as `$^!v::Send("^v")`. It is a large share of legacy rows — every registry name, alias and `vk`/`sc` code qualifies (`LegacyHotkeyFixtures`) — so downloading an untouched profile after the migration does change those users' scripts. The change is intended: without `$` a `SendKeys` binding can retrigger itself.

## Consequences

Adding an Action kind is now a schema change plus a branch in the validator, emitter, DTOs and snapshot converter, not a new convention inside a string. That cost is the point: the alternative fails at script-load time on the user's machine, where we cannot see it.

Snapshots are immutable, so pre-existing history keeps its legacy `Action`/`Parameters` members forever and the converter can never be deleted — only the live columns were dropped.
